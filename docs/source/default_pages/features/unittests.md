# unittests

Unittests are an essential quality assurance in every programming language. In python, the [unittest](https://docs.python.org/3/library/unittest.html) framework from the python standard library and the open source 3rd party [pytest](https://docs.pytest.org/en/stable/) framework are the most famous ones.  


It is arguable, how extensively unittests must be implemented in a science project but testing a defined behaviour of a function prevents that future changes result in unintended behaviour.  

While this is not a real "feature" of this Data-Science-Project-Template, the template serves a dedicated folder to write those including examples on how to write them (resp. some tests for the functionality in `utils` subpackage).  

```{Note} Key take-away
When you write unittests, write them inside the `tests` folder!
```
